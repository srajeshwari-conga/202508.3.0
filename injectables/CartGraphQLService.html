Description

The CartGraphQLService manages the cart's products and custom items that a user intends to purchase. It leverages GraphQL to fetch and update cart details such as pricing and configuration.

Usage

To inject and use the CartGraphQLService in a component:

import { CartGraphQLService, AObjectService} from '@congarevenuecloud/ecommerce';
export class MyComponent implements OnInit{
constructor( private cartService: CartGraphQLService)
}
// or in a service:
export class MyService extends AObjectService {
private cartService: CartGraphQLService = this.injector.get(CartGraphQLService);
}

Public startSSESubscription
startSSESubscription()

Initiates Server-Sent Events (SSE) subscriptions for cart pricing and configuration updates.

This method listens for real-time updates related to the cart pricing and configuration through SSE from GraphQL library. It subscribes to two separate streams:

  1. Cart Pricing Updates: The subscription receives real-time updates to cart pricing, updates the priceState, and triggers the priceCart method to process the new pricing.
  2. Configuration Updates: The subscription listens for updates to the cart configuration, processes the event if necessary, updates the configState, and triggers the getConfigStatus method to process the configuration status.

The subscriptions will be active as long as the cart ID is valid and the cart schema is provided. Errors are logged to the console and the action queue is cleared in case of configuration subscription failure.

Returns : void
Public stopSSESubscription
stopSSESubscription()

Stops the active Server-Sent Events (SSE) subscriptions for cart pricing and configuration updates. This method unsubscribes from both the cart pricing and configuration stream events from GraphQL. It ensures that no further updates are received for these events by terminating the active subscriptions.

Returns : void

getActiveCartFilters
getActiveCartFilters()

Generates an array of filters to retrieve the active cart for the user. These filters help narrow down the search for the active cart and fetch entries as needed.

This method applies a set of default filters while fetching user's active cart. However, implementors can choose to pass their own filters by extending this method. The method applies different filters based on the presence of a local cart. The filters defined by this method are as follows:

  • If a local cart exists:

    • Id: Matches the ID of the local cart stored in local storage.

  • If no local cart exists:

    • UseType: Excludes entries with a UseType of 'Shadow'.
    • BusinessObjectType: Excludes entries with a BusinessObjectType of 'FavoriteConfiguration'.

(Note: The Account.Id filter, which was previously used to refine cart record retrieval, has been removed as it was considered redundant.)

Returns : Array<FieldFilter>

An array of filters based on whether the local cart exists in the local storage.
