sendEmailByTemplate
sendEmailByTemplate(emailRequestPayload: EmailRequestPayload, contactId?: string)

Sends an email using a predefined template. You must either provide a To recipient in each wrapper or pass a contactId to auto-resolve the recipient.

If the From field is not specified, it defaults to the current logged-in (or guest) user's email and name.

If a wrapper has no To recipient and contactId is not provided, an error is thrown.

Example:

import { EmailService, EmailRequestPayload, EmailPriority } from '@congarevenuecloud/ecommerce';
import { Observable } from 'rxjs';

export class MyComponent {
  constructor(private emailService: EmailService) {}

  sendTemplateEmail() {
    const payload: EmailRequestPayload = {
      EmailTemplateId: 'template-123',
      EmailRequestWrappers: [
        {
          EmailTemplateParameters: {
            TemplateData: {
              Name: 'John Doe',
              Message: 'Interested in your services.'
            }
          },
          EmailParameters: {
            To: [
              {
                Address: 'support@test.com',
                DisplayName: 'Support Team'
              }
            ]
          }
        }
      ],
      Priority: EmailPriority.High
    };
    this.emailService.sendEmailByTemplate(payload).subscribe(success => {
    });
  }
}

Key Fields:

  • EmailTemplateId (string): ID of the email template.
  • From (EmailInfo, optional): Sender info. Defaults to current user.
  • EmailRequestWrappers (EmailTemplateWrapper[]): Each wrapper defines:
    • EmailTemplateParameters: Template merge fields (object name, record ID, etc.).
    • EmailParameters: Recipients (To, Cc, Bcc, ReplyTo).
  • Priority (EmailPriority, optional): Email priority. Default: Normal.
  • Headers (EmailHeaders, optional): Custom key-value headers.
  • ShouldProcessInbound (boolean, optional): Enable inbound email rules.
  • Attachments (EmailAttachment[], optional): List of files (name, MIME type, base64 content).

See EmailRequestPayload for more details on the structure and fields required.
